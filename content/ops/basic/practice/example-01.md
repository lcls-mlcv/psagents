# Index a local docs folder

**Goal:** Run a minimal indexing workflow on your laptop to mirror how facility docs are made searchable.

## Steps (local)

1. Create a folder with a few `.md` files.
2. Use your editor or a small script to list files and verify paths.
3. (Optional) Run a local SQLite FTS or `grep` to confirm you can query by keyword.

## What you are practicing

- Treating documentation as data you can query, not only read linearly.
- Keeping paths and roots explicit so an agent (or you) can repeat the same commands.

This is a **dummy** stand-in for a real `docs-index`-style tool.
