# observe / plan / act / verify

**Goal:** Separate phases in code so you can test each one.

## Steps (local)

1. In any language, write four functions with typed or documented inputs/outputs.
2. Add a loop that stops when `verify` sets a done flag.
3. Run with a fake `plan` that returns a no-op tool call.

## What you are practicing

The same loop structure most harnesses assume, without tying to a vendor SDK yet.
