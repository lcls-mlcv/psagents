# Sketch a five-step workflow

**Goal:** Decompose one ops workflow without writing production code yet.

## Steps (local)

1. Write steps: Intake → Retrieve → Decide → Execute → Verify → Report (drop or merge as needed).
2. For each step, name the **input type** and **output type** (even if “free text”).
3. Mark which steps must be **human-approved** if automated.

## What you are practicing

Structured workflows before you bind them to a specific harness or framework.
